## 0. jq は忘れる（参照枠は Unix）

jig の設計参照枠は **Unix**（パイプ・テキストストリーム・小さく鋭い道具）であって
jq ではない。jq からは「**JSON + CLI**」という着想だけを継いだ。

- **jq 互換は追わない**。互換が要るなら jq / gojq / jaq を使う＝他ツールの仕事。
- 前向きの設計・docs・診断・コメントは **Unix と es-toolkit の語彙**で書く。
  「jq だと…」を説明の枠にしない（移行者向けの対応は別。下記）。
- jq への言及は **歴史的参考**として [jq-compat.md](jq-compat.md)（SUPERSEDED）に集約し、
  正典ドキュメントには持ち込まない。

> 迷ったら **歴史で測る**: unix と js を比べれば分かるとおり、**長く普遍なものほど
> 価値が高い**。CLI は今でも原点にして頂点。長命で普遍な側（Unix のパイプ・テキスト
> ストリーム・小さい道具）を採る。

## 1. 一つの記号は一つの意味 — **comma は常にストリーム**

`,`（カンマ）は jig 全体で**ただ一つの意味＝ストリーム連結**（「これも出す」）。
**この不変条件を壊さない。**

- **多値が要る場所**（ソートのキー組・収集）は、その comma-ストリームを **builtin や
  構築側が food する**。例: `orderBy(.age, .name)` は「`(.age, .name)` のストリームを
  **キー組**として食う」＝ comma は依然ストリームのまま。
- **位置で役割が違う別スカラ引数**には `;` を使う。例: `range(from; to; step)`。
  ここで comma を使うと `range(1, 3)` が「ストリーム `1,3` を引数に」＝別物になるため。
- **`,` を「引数区切り」に格下げして標準化するのは悪手**。括弧の内と外で comma の
  意味が変わり（外＝ストリーム／内＝区切り）、唯一意味＝合成可能性が壊れる。
- 使い分けは機械的に決まる: **その引数位置が「ストリームを取る」のか「位置の違う
  別スカラを取る」のか**。前者は comma、後者は `;`。

---

## 2. 小さく重ねる — 合成 > 巨大ビルトイン

派生・変種は **フラグや引数を生やさず、パイプ合成**で表す。

- **降順** = `orderBy(.x) | reverse`（新しい方向引数を作らない）。
- `countBy(f)` = `groupBy(f) | mapValues(length)`。
- 新規性は**パイプでなく診断**に全振りする（§5）。パイプ自体は枯れた Unix の道具のまま。

---

## 3. 一つの `|`、一つの `.`、一つのストリーム

- **合成は `|` のみ**正典（`.a.b` 連鎖は `.a | .b` の糖衣）。
- **トピックは暗黙 `.`**。builtin 引数は素のフィルタ（`map(.name)` / `filter(.active)`）。
- **ストリームは遅延ジェネレータ**の思想（NDJSON・複数ドキュメント）。`,` は降格させ、
  レコード組立は `{…}`、ストリーム収集は `[ … ]`。

---

## 4. AI が操作する前提で予測可能に

Claude Code 等の AI が jig を**書く・直す**ことは十分ありえる。**AI にとっての予測
しやすさを最優先**する。

- **一記号一意味**（§1）→ AI は文脈で意味を切り替える判断が要らず、誤生成が減る。
- **正典は一つ**。alias（`select`=`filter` 等）は**受理するが提案しない** — docs /
  `explain` / `render`（=`fmt` の種）は正典形のみを出し、書きかけを正典へ正規化する。
- **明示 > 暗黙**、**良いエラーで復帰可能**。
- es-toolkit / JS の**既知 prior**に乗せる（名前だけ JS、文法は Unix）。

---

## 5. 診断こそ製品（footgun は黙らせない）

すべてのエラーは **span + 型名 + hint + caret**。jig の差別化はここに全振り。

- 文法は触らず、**診断層**で地雷を humane に注記する（例: `{x:(.a,.b)}` の無言 fan-out、
  `filter(u => u.active)` のアロー反射、`orderBy(.x, "desc")` の「文字列リテラルを
  ソートキーにしている＝降順なら `| reverse`」）。
- **補完・予測は診断の延長**（「現在トークンの caret 診断」→「次トークンの予測」）。
  一記号一意味・正典一つ・スキーマ駆動が、補完エンジン（`jig complete`）を最も強くする。

## 重み付け（実装で迷ったときの優先順）

> **Unix 美学 ＞ Claude フレンドリー ＞ 予測しやすさ ＞ js-like**

js-like（メソッド鎖・アロー・`|>` 等）は最下位＝後回し/任意。Unix の不変条件
（§1 comma=ストリーム・§3 一つの `|`）と衝突する JS 風糖衣は採らない。
